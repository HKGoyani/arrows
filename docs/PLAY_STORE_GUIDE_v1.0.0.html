5. Play Store Listing Copy

ASO note — no Keywords field on Play

Unlike App Store Connect, Google Play has no separate keywords field. Play's search algorithm indexes the app title, short description, and full description text directly — so, unlike the iOS listing, natural keyword density inside the description itself matters here and phrases shouldn't be needlessly avoided just because they appear elsewhere.

8. In-App Products

FieldValue
Product nameRemove Ads
Product IDremove_ads
TypeManaged product (one-time, non-consumable)
Purchase option IDremove-ads-buy
Purchase typeBuy (not Rent)
Price$3.99 USD (Google auto-converts to other currencies)
DescriptionRemove all fullscreen ads between levels. Life refill and Hint ads will still be available.

Play's current one-time product model requires at least one purchase option nested under the product — this is separate from the Product ID and is what actually carries the price/region/purchase-type config. Its ID follows the same format as subscription base plan IDs: lowercase letters, numbers, and hyphens only (no underscores), max 63 characters, must start with a letter or number. remove-ads-buy is proposed here (hyphenated, since the Product ID remove_ads already uses the underscore form and the two IDs must be visually distinguishable). The in_app_purchase Flutter package only needs the Product ID to query and purchase — the purchase option ID is a Play Console configuration detail and isn't referenced anywhere in lib/iap_service.dart.

Setup steps in Play Console

  1. Go to Monetize → Products → In-app products → Create product
  2. Enter Product ID remove_ads (must exactly match IapService.removeAdsId in lib/iap_service.dart)
  3. Add the name/description above
  4. Under the product, add a purchase option with ID remove-ads-buy, type Buy, set price to $3.99 USD equivalent and regional availability (worldwide)
  5. Activate the product
  6. Purchases can only be tested once the app is uploaded to at least the Internal testing track and the tester account is added as a license tester

This uses the in_app_purchase Flutter package with Google Play Billing on Android — the same package already used for the iOS StoreKit integration, so no separate IAP code path was needed.

9. Ads Declaration

The app shows ads via Google AdMob in 4 formats — declare "Yes, my app contains ads" in Play Console → App content → Ads:

FormatTrigger
RewardedHint (after 5 free hints), Add More Lives (lose screen)
InterstitialEvery 3rd level win, level restart, daily challenge complete
BannerBottom of Home, Challenge, Collection, Settings tabs (Anchored Adaptive format; collapsible on Home, fixed on gameplay)
App OpenApp resume from background / cold start (never during active gameplay, and suppressed on the very first session)

All ads can be permanently disabled (except rewarded, which is opt-in) via the Remove Ads IAP. Google UMP SDK consent flow (GDPR/US States) runs before any ad request, matching the iOS implementation.

13. Notes for Reviewers

Play Console doesn't have a dedicated "review notes" text field like App Store Connect, but the same information should go in the App access section (for anything requiring special instructions) and can be referenced if Google's review team reaches out:

Arrows – Escape Puzzle is a single-player offline puzzle game. No login
or account is required to play — all functionality is available
immediately from a fresh install.

- All gameplay progress is stored locally on-device (no server backend).
- Ads are served via Google AdMob using production ad units, gated by
  a Google UMP consent flow (GDPR/US States) shown before any ad request.
- One in-app purchase: "Remove Ads" ($3.99, managed product, product ID
  remove_ads) disables banner/interstitial/app-open ads. Rewarded ads
  (opt-in, for hints/extra lives) remain available even after purchase
  by design — this is disclosed in the purchase description shown to
  users in Settings.
- Firebase Analytics is used for anonymous, aggregated usage events only
  (no PII collected).
- No account creation, no chat, no user-generated content, no web view
  with unrestricted browsing.

Test path: tap Play on the Home screen to start Level 1 (tutorial),
or tap an in-progress level number to continue.
