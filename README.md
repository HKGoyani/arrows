# Arrows — native Flutter game (Android & iOS)

A faithful native port of the HTML prototype. **No WebView** — the board is drawn
with `CustomPainter`, animated with `AnimationController`/`Ticker`, and the levels
are generated by pure-Dart code.

## Run
```bash
flutter pub get
flutter run            # on a connected device / emulator / simulator
```
Builds verified:
```bash
flutter build apk --debug          # Android  ✓
flutter build ios --simulator --no-codesign   # iOS    ✓
flutter test                       # 5 tests  ✓  (40 levels solvable, boots to board)
```

## How it plays
- Tap an arrow with a clear path → it **snakes off** the board (blue, accelerating);
  the emptied cells reveal the dot grid; the progress bar advances.
- Tap a **blocked** arrow → it clashes, turns **red and stays red**, costs a life.
  3 lives → **Game Over** (Retry).
- Clear the board → **"Brilliant!"** + confetti → next level.
- Levels are procedural, deterministic, and **guaranteed solvable**; the board grows
  with the level (3 arrows on a small board → ~20 on an 11×13 board).

## Architecture (MVVM + light clean architecture)
```
lib/
  config.dart            design tokens + geometry (colors, sizes, timings)
  rng.dart               mulberry32 RNG + cubic-bezier easing (ports of the HTML)
  models.dart            Direction / Arrow / ArrowState
  level_generator.dart   winding maze generator + solvability (pure Dart)
  game_controller.dart   ChangeNotifier ViewModel: state machine + rules + hit-test
  fly_off.dart           snake "track" math for the fly-off
  board_painter.dart     CustomPainter: dots + arrows + leaving/clashed states
  confetti.dart          Ticker particle system (two bottom-corner cannons)
  widgets.dart           wordmark, top bar, hearts, progress bar, bottom nav
  intro_screen.dart      level card (auto-advances)
  game_screen.dart       board + taps + animations + win/lose overlays
  main.dart              app + intro↔game flow
```

The domain layer (`rng`, `models`, `level_generator`) has zero Flutter imports and
is unit-tested. Per-frame animations bypass the ViewModel and repaint via the painter.

## Notes
- Font: **Poppins** via `google_fonts` (fetched + cached at runtime; bundle the TTFs
  in `pubspec.yaml` if you need guaranteed-offline rendering).
- To rename the app / set an icon, edit `android/app/src/main/AndroidManifest.xml`,
  `ios/Runner/Info.plist`, and add launcher icons (e.g. `flutter_launcher_icons`).
